Клиент подключается по WebSocket для мгновенной доставки сообщений. Backend валидирует JWT-токены, сохраняет сообщения в PostgreSQL через Prisma. При отключении клиента данные остаются в БД.

Тест-кейсы в видео

1. Авторизация + отправка сообщений

2. Реал-тайм обновления и статусы

3. Тест производительности (1000 сообщений)
